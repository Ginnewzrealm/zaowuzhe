# 造物主 · Agent角色造物术

> 「不是复制人，是设计一套可运行的认知操作系统。」

## 是什么

造物主是一个用于**创建和优化 Agent 角色**的技能。

当你需要一个新角色时（比如「帮我做一个公众号运营助手」），造物主会：
1. 引导你明确角色需求
2. 调研你指定的参照人物（如：小马宋）
3. 提炼参照人物的思维框架
4. 将框架迁移到新角色
5. 生成完整的 Agent 角色提示词

## 与女娲的关系

**女娲** 是从真人蒸馏思维框架，生成「人物 Skill」用于角色扮演。
**造物主** 是从参照人物迁移思维框架，生成「完整 Agent 配置」用于创建真实 Agent。

```
女娲：真人 → 调研 → 思维框架 → 人物Skill（用于扮演）
造物主：参照人物 → 调研 → 迁移框架 → Agent配置（用于创建）
```

造物主继承了女娲的：
- 6 Agent 并行调研框架
- 三重验证筛选方法
- 表达 DNA 分析方法
- Phase 检查点机制
- 双 Agent 精炼工序

但面向全新场景：从零创建 Agent，而非从真人蒸馏。

## 核心功能

### 场景一：新建角色

| 步骤 | 说明 |
|------|------|
| Phase -1 | 环境检测（已有Agent、可用模型、工作空间） |
| Phase 0 | 需求确认（名字、职责、参照人物、性格） |
| Phase 0.5 | 创建角色目录 |
| Phase 1 | 调研参照人物（女娲6 Agent框架） |
| Phase 1.5 | 调研Review检查点 |
| Phase 2 | 框架提炼 + 迁移到新角色 |
| Phase 2.5 | 提炼确认检查点 |
| Phase 3 | 角色构建（输出完整提示词） |
| Phase 4 | 质量验证 |
| Phase 5 | 双Agent精炼 |

### 场景二：优化现有Agent

| 步骤 | 说明 |
|------|------|
| Phase 0 | 选择Agent + 确认优化方向 |
| Phase 1 | 读取现有配置，分析问题 |
| Phase 2 | 生成优化提示词 |
| Phase 3 | 二次确认执行 |

## 三种交付选项

| 选项 | 说明 | 二次确认 |
|------|------|---------|
| **A. 提示词文档** | 生成完整角色提示词文件包 | ✅ |
| **B. 直接创建指令** | 连接OpenClaw直接创建新Agent | ✅ 确认执行 |
| **C. 优化提示词** | 生成发给指定Agent的提示词，让它自我优化 | ✅ 确认发给谁+确认内容 |

## 设计理念

一个好的 Agent 角色是一套可运行的认知操作系统：

| 维度 | 说明 |
|------|------|
| **心智模型** | 他用什么镜片看世界？ |
| **决策启发式** | 他用什么快速规则做判断？ |
| **表达DNA** | 他怎么说话？ |
| **反模式** | 他绝对不会做什么？ |
| **诚实边界** | 什么是这个角色做不到的？ |
