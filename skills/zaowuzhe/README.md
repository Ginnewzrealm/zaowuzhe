# 造物者 · Agent 角色设计师

> 想创建一个 Agent 角色，但网上找的 Prompt 都不满意？

---

## 为什么会创建这个技能

我们一直在用 **女娲** 技能——它能从真人蒸馏思维框架，生成一个可用的人物 Skill，非常好用。

但有时候我们想要的不只是一个"技能"，而是一个**完整的 Agent 角色**：
- 有独特的个性表达
- 有明确的行为规范（Red Lines、权限控制）
- 有记忆系统和心跳任务
- 可以绑定飞书/Telegram 对话
- 能在 OpenClaw 里独立运行

于是我们在女娲的基础上，改出了"造物者"——专门用来创建 Agent 角色和人格设定的技能。

---

## 问题场景

- 想创建一个 Agent 角色，但网上的模板太通用、不够有灵魂
- 需要一个专业领域的 Agent（投资顾问、运营专家、写作助手），但不知道怎么定义它的"思维方式"
- 希望角色有独特的个性和方法论，而不是千篇一律的"你好，我是XXX助手"
- 女娲生成的 Skill 很好，但想把它升级成一个真正的 Agent 角色

---

## 解决方案

**造物者**通过**蒸馏参照人物的思维框架**来创建 Agent 角色。

就像女娲从真人蒸馏思维框架一样，造物者让用户指定一个"参照人物"（如：小马宋、芒格、马斯克），然后：
1. 调研这个人物的核心思维方法和决策模式
2. 把这些方法论迁移到新角色上
3. 结合标准的 Agent 规范，生成完整可用的角色提示词

---

## 核心特点

| 特点 | 说明 |
|------|------|
| **继承女娲精华** | 复用女娲的 6 Agent 调研框架，确保参照人物的思维被充分挖掘 |
| **升级为 Agent** | 不是生成 Skill，而是生成完整的 Agent 配置（SOUL.md、AGENTS.md、IDENTITY.md 等） |
| **SOP 问询引导** | 不是填空，而是引导你思考角色定位、性格、能力边界 |
| **三种交付方式** | 提示词文档 / 直接创建 / 优化现有 Agent |
| **环境感知** | 自动检测 OpenClaw 环境，匹配已有配置 |
| **二次确认** | 所有重要决策都需要用户确认，避免创建出不符合预期的角色 |

## 效果实例

### 实例一：创建一个公众号运营角色

**用户输入**：
> 我想做一个公众号运营角色，参照人物是小马宋

**造物者引导**：
```
检测到当前环境：
- 已有 Agent: mira, maxiu, angie, rednote, wen, jinjie
- 可用模型: Gemma 4 E4B (本地/免费), Kimi K2.5, MiniMax M2.7

请确认以下配置项：
1. Agent ID：____________（默认：gzh-operater）
2. 显示名：____________（默认：公众号运营）
3. Workspace 路径：____________（默认：~/Documents/Openclaw-obsidian/Obsidian Vault/gzh-operater）
4. 创建位置：Claude Code / OpenClaw
```

**交付产物**：
- SOUL.md（角色灵魂定义）
- IDENTITY.md（身份标识）
- USER.md（用户信息）
- AGENTS.md（工作空间规则）
- HEARTBEAT.md（心跳任务）
- MEMORY.md（长期记忆）
- HIPPOCAMPUS_CORE.md（记忆核心）

### 实例二：优化现有 Agent

**用户输入**：
> 帮我优化一下 mira 的设定

**造物者分析**：
```
【mira 当前设定分析】

优点：
- 身份定位清晰
- 有完整的记忆系统

问题：
- 性格描述过于模糊
- 禁忌清单不够具体

建议优化：
1. 性格描述：从"专业"改为更具体的"直接果断、有观点"
2. 禁忌清单：补充"不外泄私人数据"等通用禁忌
...

确认执行？每条修改都需要您明确回复「确认」或「跳过」。
```

## 工作原理

### 流程图

```
用户输入：我想做一个XX角色
         ↓
┌─────────────────────────────────────┐
│ Phase -1: 环境检测                  │
│ - 检测 OpenClaw 已有 Agent          │
│ - 检测可用模型                      │
│ - 检测 IM 渠道绑定                  │
└─────────────────────────────────────┘
         ↓
┌─────────────────────────────────────┐
│ Phase 0: 需求分流                   │
│ - 新建角色 → 进入场景一              │
│ - 优化现有 → 进入场景二              │
└─────────────────────────────────────┘
         ↓
┌─────────────────────────────────────┐
│ 场景一：新建角色                    │
│ Phase 0.1: 基本信息确认             │
│ Phase 0.2: 创建位置确认（二次确认）  │
│ Phase 1: 调研参照人物（女娲6 Agent）│
│ Phase 2: 框架提炼 + 迁移             │
│ Phase 3: 角色构建                   │
│ Phase 4: 质量验证                   │
│ Phase 5: 双Agent精炼                │
└─────────────────────────────────────┘
         ↓
┌─────────────────────────────────────┐
│ 三种交付方式                        │
│ A. 提示词文档（生成 MD 文件）       │
│ B. 直接创建（连接 OpenClaw）        │
│ C. 优化提示词（发给现有 Agent）     │
└─────────────────────────────────────┘
```

### 女娲框架的继承

造物者在 **Phase 1** 完全复用女娲的 6 Agent 并行调研框架：

| Agent | 调研维度 |
|-------|---------|
| Agent1 | 著作与系统性思考 |
| Agent2 | 对话与即兴思维 |
| Agent3 | 表达DNA |
| Agent4 | 他者视角 |
| Agent5 | 决策记录 |
| Agent6 | 时间线 |

这确保了参照人物的思维框架被全面、深度的挖掘。

## 创建角色应考虑的维度

基于 OpenClaw AGENTS.md 规范，造物者会系统考虑以下维度：

| 维度 | 说明 |
|------|------|
| **身份定位** | 名字、角色、核心使命 |
| **性格特质** | 说话风格、决策倾向 |
| **表达方式** | 语气、句式、幽默感 |
| **行为规则** | 什么能做、什么不能做 |
| **知识边界** | 擅长领域、禁区 |
| **工具/Skills** | 需要哪些技能 |
| **记忆系统** | 长期记忆如何积累 |
| **沟通渠道** | 绑定哪些 IM（飞书/Telegram） |
| **工作空间** | 文件目录结构 |
| **权限设定** | 操作权限、隔离级别 |

## 背后的故事

造物者的诞生源于一次实践：

我们在使用女娲技能时，发现它生成的"人物 Skill"非常精准——能抓住一个人的思维本质。但女娲的输出是一个 Skill，而不是一个能独立运行的 Agent。

如果我们想要：
- 让这个角色有记忆
- 能绑定 Telegram 对话
- 有心跳任务能主动工作
- 在 OpenClaw 里作为独立 Agent 运行

就需要把 Skill 升级成完整的 Agent 配置。

于是造物者诞生了——它复用女娲的调研能力，但输出的是完整的 Agent，而不是 Skill。

**造物者，让每个想法都能被塑造成有灵魂的 Agent。**
